Quand TestDisk est exécuté, vous pouvez voir l'expression Please wait ... (Attendez s'il vous plaît) sur votre écran jusqu'à ce qu'il ait recueilli assez de données du BIOS ou de l'OS pour énumérer les unités de disques sur le système.

TestDisk is a data recovery designed to help recover lost partitions
and/or make non-booting disks bootable again when these symptoms
are caused by faulty software, certain types of viruses or human error.
It can also be used to repair some filesystem errors.

Information gathered during TestDisk use can be recorded for later
review. If you choose to create the text file, <b>testdisk.log</b> , it
will contain TestDisk options, technical information and various
outputs; including any folder/file names TestDisk was used to find and
list onscreen.

Use arrow keys to select, then press Enter key:
<b>[ Create ]</b>  Create a new log file
[ Append ]  Append information to log file
[ No Log ]  Don't record anything

TestDisk propose d'enregistrer les informations collectées durant son exécution pour analyse ultérieure. Si demandé, il créé un fichier texte nommé testdisk.log.

Select a media (use Arrow keys, then press ENTER):
Disk /dev/hdc - 708 MB / 675 MiB
Disk /dev/sda - 120 GB / 111 GiB
Disk /dev/sdb - 120 GB / 111 GiB





<b>[Proceed ]</b>  [  Quit  ]


Note: Disk capacity must be correctly detected for a successful recovery.
If a disk listed above has incorrect size, check HD jumper settings, BIOS
detection, and install the latest OS patches and disk drivers.


Si la taille rapportée n'est pas égale à la taille du disque dur, c'est à dire, un disque dur de 120 Go est identifié comme étant seulement un disque dur de 32 Go, vérifier les paramètres du disque dur dans le BIOS et les cavaliers (jumpers) sur le disque. Sur la plupart des disques durs, il y a des cavaliers (jumpers) pour limiter la taille à seulement 8 ou 32 Go. Si votre disque dur est détecté comme étant de 130 Go seulement, le support LBA48 ne doit pas être disponible dans votre OS, lire Notes spécifiques aux systèmes d'exploitation pour plus d'informations.


La prochaine étape est de choisir le type de table de partition.

Eléments du Menu TestDisk

  • Analyse Analyse la partition courante et cherche les partitions perdues
  • Menu Advanced, Utilitaires de réparation de système de fichiers
    • FAT: Réparation du secteur de boot et de la FAT
    • NTFS: Réparation du secteur de Boot et de la MFT
    • EXT2/EXT3: Recherche des SuperBlocs de secours
  • Geometry Modifier la géometrie du disque
  • Options Modifier les options
  • MBR Code Ecrire le code du MBR de TestDisk sur le premier secteur
  • Delete Effacer toutes les données dans la table de partition

Si vous ne comprenez pas comment utiliser TestDisk

  • lancez testdisk_win ou testdisk /log /debug (assurez-vous qu'il y ait un espace avant chaque slash),
  • choisissez le disque dur défectueux en utilisant les touches directionnelles puis pressez la touche ENTREE,
  • une fois TestDisk terminé (Note: vous aurez peut-être besoin de presser la touche ENTREE 2/3 fois pendant le traitement), choisissez Search! pour redémarrer l'analyse.
  • envoyez simplement le fichier que TestDisk a créé, testdisk.log, à grenier@cgsecurity.org avec une brève explication du problème et les caractéristiques des partitions précédentes (taille, nom de volume, système de fichiers).
