Monitoreo SMART

From CGSecurity

Jump to: navigation, search

Es posible controlar y monitorear los sistemas de almacenamiento usando el Self-Monitoring, Analysis and Reporting Technology System (SMART) incluídos los más modernos ATA y SCSI discos rígidos.

Con el objetivo de anticipar fallas, SMART monitorea y reporta varios indicadores de confiabilidad incluyendo la temperatura, cantidad de sectores reubicados y errores de búsqueda.

Algunos consejos:

  • Habilite el monitoreo SMART en el setup de su BIOS. Con SMART habilitado, si una operación de escritura falla, los sectores dañados son puestos fuera de servicio y los datos son inmediatamente escritos a otro sector. En otras palabras, el sector dañado es reubicado.
  • Utilice un software para reportar la información de SMART, de esta manera tendrá detección temprana de problemas físicos.
    • smartmontools — open-source para Windows, Linux, FreeBSD, NetBSD, OpenBSD, Solaris, Darwin, OS/2.
    • SMARTReporter — open-source para Apple Macintosh
  • Si su disco rígido está conectado via USB ó FireWire, el status de SMART puede no ser reportado, si este es el caso, es preferible una conexión directa usando IDE, SCSI, SATA...

Precaución en cuanto a SMART

SMART puede no funcionar igualmente bien para todos los discos rígidos en los cuales ha sido instalado y habilitado. Sugerimos que lleve adelante algunas pruebas usando este sistema (quizás para sus discos específicos) antes de decidir cuanto puede confiar en SMART. Varios fabricantes de discos rígidos parecen tener variadas ideas en que incluir ó excluir del monitoreo SMART, y entre los datos que los distintos discos proveen, puede haber diferentes "seteos de umbrales (threshold settings)" para decidir que y cuando SMART debe informar que hubo cambios en un disco.
