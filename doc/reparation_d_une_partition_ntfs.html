Réparation du secteur de boot NTFS

Si le secteur de boot NTFS est endommagé, les données ne sont plus accessibles. Windows affiche: The drive is not formatted, do you want to format it now? "Le pilote n'est pas formaté, souhaitez-vous le formater maintenant?" Linux affichera: wrong fs type, bad option, bad superblock "Mauvais type fs, mauvaise option, mauvais superbloc".

TestDisk vous permet de manipuler les secteurs de boot des partitions NTFS. Sélectionnez la partition que vous souhaitez modifier et choisissez Boot. Si le secteur de boot NTFS et le secteur de boot de sauvegarde ne sont pas identiques, vous pourrez recopier le secteur de boot de sauvegarde dans le secteur de boot et vice versa. Vous pouvez aussi reconstruire le secteur de boot NTFS ou vider le contenu du secteur. Pour reconstruire le secteur de boot NTFS, TestDisk essaie de localiser la MFT (Master File Table: $MFTMirr). Il lit la taille de la MFT et calcule la taille du cluster. Il lit la taille de l'Index Allocation Entry dans l'index du répertoire root. En utilisant toutes ces valeurs, il peut fournir un nouveau secteur de boot. TestDisk laisse l'utilisateur lister les fichiers avant d'écrire.

Réparation de la MFT d'une partition NTFS

La MFT (Master File Table) se retrouve parfois endommagé et l'utilitaire Microsoft Check Disk (chkdsk) échoue parfois dans sa réparation. TestDisk peut tenter une réparation en utilisant la MFT mirror, la sauvegarde de la MFT. Dans le menu Advanced de TestDisk, sélectionnez la partition NTFS et choisissez "Repair MFT".
