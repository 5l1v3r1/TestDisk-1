set DJGPP=D:\gnu\djgpp.env
set PATH=D:\GNU\BIN;%PATH%

DJGPP korrigieren

In include/signal.h, füge hinzu 

#ifndef SIGCONT
#define SIGCONT 18
#endif

In include/mntent.h, füge hinzu 

#ifndef MOUNTED
#define MOUNTED "/etc/mtab"
#endif
#ifndef MNTOPT_RO
#define MNTOPT_RO "ro"
#endif

In include/errno.h, füge hinzu 

#ifndef EMSGSIZE
#define EMSGSIZE        90
#endif
#ifndef EOPNOTSUPP
#define EOPNOTSUPP      95
#endif
#ifndef ENOTSUPP
#define ENOTSUPP        524
#endif
#ifndef ENOTSUP
#define ENOTSUP         524
#endif
#ifndef ECANCELED
#define ECANCELED       125
#endif
#ifndef EILSEQ
#define EILSEQ          84
#endif
#ifndef EOVERFLOW
#define EOVERFLOW       600
#endif

In include/fcntl.h, füge hinzu 

#define O_LARGEFILE     0100000

In sys/param.h, füge hinzu 

/* Some autoconf'd packages check for endianness.  When cross-building we
   can't run programs on the target.  Fortunately, autoconf supports the
   definition of byte order in sys/param.h (that's us!).
   The values here are the same as used in gdb/defs.h (are the more
   appropriate values?).  */
#define BIG_ENDIAN      4321
#define LITTLE_ENDIAN   1234
/* All known dos systems are little endian.  */
#define BYTE_ORDER      LITTLE_ENDIAN

Füge der Datei include/stdint.h hinzu 

typedef unsigned char uint8_t;
typedef unsigned short int uint16_t;
typedef unsigned int uint32_t;
typedef unsigned long long int uint64_t;
typedef char int8_t;
typedef short int int16_t;
typedef int int32_t;
typedef long long int int64_t;

ln -s gpp.exe g++.exe

TestDisk ist ein echtes 32-bit-Programm (kompiliert unter DJGPP), weshalb ein DPMI-Server-Programm (cwsdpmi.exe) beim Herunterladen mit einbezogen ist, das es ermöglicht, dass TestDisk unter 16-Bit DOS ausgeführt werden kann. Wenn du unter realen (16-Bit) DOS arbeiten möchtest, mußt du einen laufenden DPMI-Server haben; um das zu tun, erhalte: csdpmi*b.zip und platziere die ausführbare Datei (cwsdpmi.exe) einfach in dein Verzeichnis zu den Binärdateien (welche sich im Pfad befinden muß).
